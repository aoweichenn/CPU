\chapter{Linux、工具链、调试器和学习工作流}

\section{为什么需要工具基础}

底层学习不是只读概念。你必须能编译、反汇编、调试、测量、保存结果和复现实验。本章建立最低工具能力。

\section{基本命令}

必须熟悉：

\begin{itemize}
  \item \code{g++}、\code{clang++}：编译。
  \item \code{objdump}：反汇编。
  \item \code{readelf}：查看 ELF。
  \item \code{nm}：查看符号。
  \item \code{gdb}：调试和单步。
  \item \code{lscpu}：查看 CPU。
  \item \code{time}、\code{perf}：基础测量。
\end{itemize}

\section{构建类型}

Debug、Release、sanitizer、coverage build 目的不同。Debug build 不用于性能结论；Release build 不替代 correctness 测试。

\section{GDB 最小工作流}

\begin{lstlisting}[language=bash]
g++ -O0 -g main.cpp -o app
gdb -q ./app
\end{lstlisting}

常用命令：

\begin{lstlisting}
break main
run
next
step
stepi
info registers
x/10i $rip
disassemble /m main
\end{lstlisting}

\section{实验记录}

每次实验至少记录：

\begin{itemize}
  \item 日期。
  \item CPU 和 OS。
  \item 编译器版本。
  \item 编译 flags。
  \item 输入规模。
  \item 命令。
  \item 原始输出。
  \item 结论。
\end{itemize}

\section{习题}

\begin{exercisebox}
选择一个三行 C++ 函数，分别用 \code{g++ -O0 -g} 和 \code{g++ -O3} 编译。用 GDB 单步 Debug 版本，用 \code{objdump} 阅读 Release 版本。写一页报告说明两者差异。
\end{exercisebox}
