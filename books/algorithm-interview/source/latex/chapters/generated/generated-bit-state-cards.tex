\section{位运算与状态压缩：题型分流与案例}
这一节先把同一题型的常见场景拆开讲清楚，再进入具体题目。学习顺序是：先写暴力基线，定位最贵的重复工作；再根据题目条件选择能维护不变量的数据结构；最后用多个案例比较边界、反例和变体。

\begin{principlebox}{题型分流：位运算与状态压缩}
\textbf{异或抵消。}
判断条件：除一个或两个数外，其他数按偶数次出现。处理方式：利用异或交换结合和 x 异或 x 为零。代表案例：136 只出现一次的数字、260 只出现一次的数字 III。风险点：出现次数不是两次时不能直接异或。

\textbf{按位计数。}
判断条件：其他数字出现固定 k 次，目标数字出现不同次数。处理方式：逐位统计一的数量，对 k 取模还原答案。代表案例：137 只出现一次的数字 II、剑指 56 数组中数字出现次数。风险点：负数和符号位要使用足够宽的整数。

\textbf{掩码集合。}
判断条件：状态是若干布尔选择或访问集合。处理方式：用整数位表示元素是否已选、资源是否拥有。代表案例：78 子集、90 子集 II、864 获取所有钥匙的最短路径。风险点：同一位置不同掩码不能合并。

\textbf{位运算算术和低位技巧。}
判断条件：题目要求不用普通算术或快速枚举子集。处理方式：用 lowbit、子集枚举或位加法模拟状态变化。代表案例：191 位 1 的个数、338 比特位计数、371 两整数之和。风险点：移位优先级和有符号溢出需要谨慎。

\end{principlebox}

本节案例覆盖 LeetCode 29 两数相除、LeetCode 136 只出现一次的数字、LeetCode 137 只出现一次的数字 II、LeetCode 190 颠倒二进制位、LeetCode 191 位 1 的个数、LeetCode 201 数字范围按位与、LeetCode 231 2 的幂、LeetCode 260 只出现一次的数字 III、LeetCode 268 丢失的数字、LeetCode 287 寻找重复数、LeetCode 318 最大单词长度乘积、LeetCode 338 比特位计数、LeetCode 371 两整数之和、LeetCode 393 UTF-8 编码验证、LeetCode 421 数组中两个数的最大异或值。阅读时不要按题号背答案，而要先判断题面落在哪个分流场景：查询目标是什么，输入约束给了什么单调性或等价关系，暴力慢在哪里，优化结构保存了什么状态。

\subsection{代表案例与单点分析}

\subsubsection{LeetCode 29 两数相除}

\textbf{题目模型。} 除法可以看成不断减去除数的倍增值。

\textbf{推导重点。} 把除数按二进制倍增，优先减去不超过被除数的最大倍数并累加商。

\textbf{边界。} 除数为一、被除数为 int 最小值、结果溢出、符号处理。

\textbf{变体追问。} 它和快速幂类似，都利用二进制拆分减少重复加减。

\subsubsection{LeetCode 136 只出现一次的数字}

\textbf{题目模型。} 成对元素可以用异或抵消，剩下单个元素。

\textbf{推导重点。} 异或满足交换结合，扫描顺序不影响答案。

\textbf{边界。} 零、负数、唯一值在开头或结尾、所有其他值恰好两次。

\textbf{变体追问。} 若其他值出现三次，要改为按位计数取模。

\subsubsection{LeetCode 137 只出现一次的数字 II}

\textbf{题目模型。} 除一个数字外，其余数字都出现三次，单纯异或不能抵消。

\textbf{推导重点。} 逐位统计一的数量，对三取模后还原目标数字。

\textbf{边界。} 负数、符号位、目标为零、所有其他数恰好三次。

\textbf{变体追问。} 若其他数出现 k 次，思路仍是按位计数取模。

\subsubsection{LeetCode 190 颠倒二进制位}

\textbf{题目模型。} 每一位的新位置由原位置镜像决定。

\textbf{推导重点。} 循环三十二次，把结果左移并接上当前最低位，再右移输入。

\textbf{边界。} 最高位、最低位、全零、全一、无符号语义。

\textbf{变体追问。} 若多次调用，可以按字节预处理反转表。

\subsubsection{LeetCode 191 位 1 的个数}

\textbf{题目模型。} 每次清掉最低位的一可以让循环次数等于一的个数。

\textbf{推导重点。} 使用 n \& (n - 1) 删除最低位一，比逐位检查更直接。

\textbf{边界。} 零、只有最高位、一的个数很多、无符号输入。

\textbf{变体追问。} 比特位计数可以把这个过程 DP 化到一整段数。

\subsubsection{LeetCode 201 数字范围按位与}

\textbf{题目模型。} 区间内所有数按位与后，只保留左右端公共二进制前缀。

\textbf{推导重点。} 不断右移 left 和 right，直到二者相等，再把公共前缀移回原位。

\textbf{边界。} left 等于 right、区间跨过二的幂边界、结果为零。

\textbf{变体追问。} 也可以不断清除 right 的最低位一，直到 right 不大于 left。

\subsubsection{LeetCode 231 2 的幂}

\textbf{题目模型。} 二的幂在二进制中只有一个一。

\textbf{推导重点。} 正数 n 满足 n 与 n 减一按位与为零。

\textbf{边界。} n 为零、负数、int 最小值、最高位。

\textbf{变体追问。} 判断四的幂还要额外限制一所在的位置为偶数位。

\subsubsection{LeetCode 260 只出现一次的数字 III}

\textbf{题目模型。} 两个只出现一次的数异或后至少有一位不同。

\textbf{推导重点。} 用最低不同位把数组分成两组，每组内部再异或抵消。

\textbf{边界。} 两个答案一正一负、最低位分组、其他数恰好两次。

\textbf{变体追问。} 它是在异或抵消基础上增加分组位。

\subsubsection{LeetCode 268 丢失的数字}

\textbf{题目模型。} 下标零到 n 与数组值相比，缺失值会在异或抵消后剩下。

\textbf{推导重点。} 把所有下标和所有值一起异或，重复出现的数抵消。

\textbf{边界。} 缺失零、缺失 n、数组长度一、也可用求和但要防溢出。

\textbf{变体追问。} 异或法适合展示值域完整且只有一个缺口的结构。

\subsubsection{LeetCode 287 寻找重复数}

\textbf{题目模型。} 数组值域使下标到值形成函数图，重复数是环入口。

\textbf{推导重点。} 快慢指针不修改数组且常数空间；值域二分用抽屉原理。

\textbf{边界。} 重复数出现多次、不能排序、不能改数组、长度为 n 加一。

\textbf{变体追问。} 快慢指针要求值都能作为合法下标。

\subsubsection{LeetCode 318 最大单词长度乘积}

\textbf{题目模型。} 两个单词没有公共字母时才可相乘，字母集合可压成位掩码。

\textbf{推导重点。} 每个单词生成 26 位掩码，两个掩码按位与为零表示无交集。

\textbf{边界。} 重复字母、空字符串、多个单词同掩码、乘积范围。

\textbf{变体追问。} 若字符集大于整数位数，需要 bitset 或哈希集合。

\subsubsection{LeetCode 338 比特位计数}

\textbf{题目模型。} 每个数的一的个数可由去掉最低位一后的较小数转移。

\textbf{推导重点。} dp[x] = dp[x \& (x - 1)] + 1，或 dp[x] = dp[x >> 1] + (x \& 1)。

\textbf{边界。} n 为零、二的幂、连续区间、表达式优先级。

\textbf{变体追问。} 它展示位运算和 DP 的结合，而不是逐个数循环数位。

\subsubsection{LeetCode 371 两整数之和}

\textbf{题目模型。} 不用加号时，加法可以拆成无进位和与进位。

\textbf{推导重点。} 异或得到无进位和，与运算后左移得到进位，迭代直到进位为零。

\textbf{边界。} 正负数、进位传播、有符号溢出、需要使用无符号中间值。

\textbf{变体追问。} 这是位级算术模拟，和异或抵消类题目的语义不同。

\subsubsection{LeetCode 393 UTF-8 编码验证}

\textbf{题目模型。} 每个字节的高位模式决定一个字符需要的后续字节数量。

\textbf{推导重点。} 用位掩码判断首字节类型，并维护还需要多少个 continuation 字节。

\textbf{边界。} 非法首字节、后续字节不足、后续字节格式错误、单字节字符。

\textbf{变体追问。} 这是位模式解析题，重点是协议规则和状态机。

\subsubsection{LeetCode 421 数组中两个数的最大异或值}

\textbf{题目模型。} 最大异或值由高位到低位尽量取一决定。

\textbf{推导重点。} 逐位尝试把当前前缀答案设为一，用哈希集合验证是否存在两个前缀配对。

\textbf{边界。} 零、重复数、最高位、负数语义。

\textbf{变体追问。} 也可以建二进制 Trie，每个数尽量走相反位。
