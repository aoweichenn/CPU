\section{回溯、枚举与剪枝：题型分流与案例}
这一节先把同一题型的常见场景拆开讲清楚，再进入具体题目。学习顺序是：先写暴力基线，定位最贵的重复工作；再根据题目条件选择能维护不变量的数据结构；最后用多个案例比较边界、反例和变体。

结构选择 & 剪枝来自容量、剩余长度、排序后去重、合法性预检查和提前终止。组合题关注起点推进，排列题关注元素是否使用，分割题关注当前片段是否合法。\\

\begin{principlebox}{题型分流：回溯、枚举与剪枝}
\textbf{子集和组合。}
判断条件：答案不关心顺序，元素按起点向后选择。处理方式：递增起点防止重复排列，同层去重防止重复值重复生成。代表案例：78 子集、90 子集 II、77 组合、216 组合总和 III。风险点：同层去重和路径去重不是一回事。

\textbf{排列。}
判断条件：答案关心顺序，每个位置选择一个未使用元素。处理方式：used 数组记录路径中已使用对象。代表案例：46 全排列、47 全排列 II。风险点：重复元素排列要排序并跳过同层等价选择。

\textbf{分割和构造。}
判断条件：每一步选择一个前缀、片段或局部结构。处理方式：检查片段合法性后进入下一位置。代表案例：131 分割回文串、93 复原 IP 地址、140 单词拆分 II。风险点：输出规模可能指数级，复杂度要包含输出。

\textbf{棋盘和约束搜索。}
判断条件：选择会影响行列、对角线或局部访问状态。处理方式：用集合或位掩码保存约束，进入退出时成对修改。代表案例：51 N 皇后、79 单词搜索、212 单词搜索 II。风险点：剪枝只能删除不可能成功的分支。

\end{principlebox}

本节案例覆盖 LeetCode 17 电话号码的字母组合、LeetCode 22 括号生成、LeetCode 39 组合总和、LeetCode 40 组合总和 II、LeetCode 46 全排列、LeetCode 47 全排列 II、LeetCode 51 N 皇后、LeetCode 78 子集、LeetCode 79 单词搜索、LeetCode 90 子集 II、LeetCode 93 复原 IP 地址、LeetCode 131 分割回文串、LeetCode 212 单词搜索 II、LeetCode 216 组合总和 III、LeetCode 698 划分为 k 个相等的子集。阅读时不要按题号背答案，而要先判断题面落在哪个分流场景：查询目标是什么，输入约束给了什么单调性或等价关系，暴力慢在哪里，优化结构保存了什么状态。

\subsection{代表案例与单点分析}

\subsubsection{LeetCode 17 电话号码的字母组合}

\textbf{题目模型。} 每个数字对应若干字母，答案是所有按位选择形成的字符串。

\textbf{推导重点。} 暴力就是完整笛卡尔积，优化重点是路径长度和下标同步推进，避免在每层重新拼接大量中间字符串。

\textbf{边界。} 空输入、包含 7 或 9、输入中不应出现 0 和 1、输出规模指数增长。

\textbf{变体追问。} 若字符映射来自键盘以外的字典，只需要替换候选表，搜索骨架不变。

\subsubsection{LeetCode 22 括号生成}

\textbf{题目模型。} 合法括号串的前缀中右括号数量不能超过左括号数量。

\textbf{推导重点。} 状态保存已用左括号和右括号数量，只有满足约束的选择才继续递归。

\textbf{边界。} n 为零或一、右括号提前超过左括号、输出规模是卡特兰数。

\textbf{变体追问。} 若有多种括号类型，状态还需要栈保存类型匹配关系。

\subsubsection{LeetCode 39 组合总和}

\textbf{题目模型。} 候选数字可重复使用，答案不关心排列顺序。

\textbf{推导重点。} 排序后按起点枚举，递归时仍传当前下标表示可以继续使用同一数字。

\textbf{边界。} 目标小于最小数、刚好等于、候选有重复时题意如何处理、输出规模。

\textbf{变体追问。} 组合总和 II 每个数只能用一次，起点推进和去重条件都不同。

\subsubsection{LeetCode 40 组合总和 II}

\textbf{题目模型。} 每个候选只能使用一次，输入中可能存在重复值。

\textbf{推导重点。} 排序后递归下一层从 i 加一开始，同层遇到相同值要跳过。

\textbf{边界。} 全重复、目标为零、同层去重写成跨层去重、答案顺序不要求。

\textbf{变体追问。} 子集 II 使用同样的同层去重，只是每个路径节点都可能成为答案。

\subsubsection{LeetCode 46 全排列}

\textbf{题目模型。} 排列关心元素顺序，每个位置从所有未使用元素中选择一个。

\textbf{推导重点。} 路径长度达到 n 时产生答案，used 数组保证同一元素不重复使用。

\textbf{边界。} 空数组、单元素、输出规模为 n 阶乘、元素值唯一假设。

\textbf{变体追问。} 若有重复元素，需要排序并在同层跳过等价选择。

\subsubsection{LeetCode 47 全排列 II}

\textbf{题目模型。} 重复元素会让不同下标选择生成相同排列。

\textbf{推导重点。} 排序后使用 used 数组，同层遇到相同值且前一个相同值未使用时跳过。

\textbf{边界。} 全重复、部分重复、同层去重条件写反、输出顺序不要求。

\textbf{变体追问。} 子集 II 和组合总和 II 也是同层去重，但起点推进方式不同。

\subsubsection{LeetCode 51 N 皇后}

\textbf{题目模型。} 每一行放一个皇后，列和两条对角线不能冲突。

\textbf{推导重点。} 逐行搜索，用列集合、主对角线集合和副对角线集合快速判断合法性。

\textbf{边界。} n 为一、n 为二或三无解、对角线编号、回退时状态恢复。

\textbf{变体追问。} 可用位掩码压缩三个约束集合，提高常数性能。

\subsubsection{LeetCode 78 子集}

\textbf{题目模型。} 每个元素都有选或不选两种选择，答案是所有路径节点。

\textbf{推导重点。} 回溯按起点向后枚举，避免不同顺序生成同一集合。

\textbf{边界。} 空数组、单元素、输出数量为二的 n 次方、顺序不要求。

\textbf{变体追问。} 也可用位掩码枚举所有子集，适合 n 较小的场景。

\subsubsection{LeetCode 79 单词搜索}

\textbf{题目模型。} 网格路径不能重复使用同一格，状态包含位置、匹配下标和访问标记。

\textbf{推导重点。} 剪枝包括首字符不匹配、剩余长度不足、字符频次不足和越界访问。

\textbf{边界。} 单字符单词、路径需要回退、重复字母、单元格不能复用。

\textbf{变体追问。} 若要查很多单词，应使用 Trie 合并搜索前缀。

\subsubsection{LeetCode 90 子集 II}

\textbf{题目模型。} 重复元素会让不同下标路径生成相同集合。

\textbf{推导重点。} 排序后在同一层跳过相同值，保留不同层使用重复值的可能。

\textbf{边界。} 全重复、空集、重复值相邻、去重条件写成同层而不是全局。

\textbf{变体追问。} 组合总和 II 使用同样的同层去重思想。

\subsubsection{LeetCode 93 复原 IP 地址}

\textbf{题目模型。} 字符串要切成四段，每段必须是 0 到 255 且无非法前导零。

\textbf{推导重点。} 按段数和剩余长度剪枝，每次尝试长度一到三的片段。

\textbf{边界。} 前导零、数值超过 255、剩余字符太多或太少、刚好四段。

\textbf{变体追问。} 分割回文串也是前缀分割，但合法性检查不同。

\subsubsection{LeetCode 131 分割回文串}

\textbf{题目模型。} 每次选择一个从当前位置开始的回文前缀。

\textbf{推导重点。} 预处理回文 DP 表，让回溯合法性检查为常数。

\textbf{边界。} 空串、单字符、全相同字符、重复分割路径。

\textbf{变体追问。} 若只求最少切割次数，可改成 DP 最短切分。

\subsubsection{LeetCode 212 单词搜索 II}

\textbf{题目模型。} 多个单词在同一棋盘上搜索，公共前缀可以共享。

\textbf{推导重点。} 先建 Trie，再从每个格子回溯；路径到达单词结束节点就收集答案。

\textbf{边界。} 重复单词、同一格不能复用、前缀是完整单词、剪枝删除空子树。

\textbf{变体追问。} 这是单词搜索从单目标扩展到多目标后的结构升级。

\subsubsection{LeetCode 216 组合总和 III}

\textbf{题目模型。} 从一到九中选 k 个数和为 n，每个数最多用一次。

\textbf{推导重点。} 按递增起点枚举，利用剩余个数和剩余和剪枝。

\textbf{边界。} n 太小、n 太大、k 为零、路径长度已经超过 k。

\textbf{变体追问。} 可预估剩余最小和最大和进一步剪枝。

\subsubsection{LeetCode 698 划分为 k 个相等的子集}

\textbf{题目模型。} 每个元素要分配到 k 个桶中，所有桶目标和相同。

\textbf{推导重点。} 先判断总和可整除并排序降序，再用桶剩余容量回溯放置元素。

\textbf{边界。} 存在元素大于目标、重复元素、空桶对称、k 为一。

\textbf{变体追问。} 状态压缩 DP 也能做，适合用掩码表示已选元素集合。
