\chapter{数组、双指针和滑动窗口}

数组题是刷题的地基，但这句话不能被理解成“数组题就是双指针和滑动窗口”。真正的学习顺序应该是：先理解数组本体，知道连续内存、下标、区间、扫描和原地写入分别意味着什么；再做一组纯数组题，把前缀、差分、前后缀、覆盖写入这些基础动作练扎实；然后才进入双指针，因为双指针是在数组候选空间上做淘汰；最后进入滑动窗口，因为窗口是在连续区间上维护增量状态。顺序反过来，就会出现一个很常见的问题：代码看起来像模板，解释时却说不清为什么能移动某个指针。

本章按这个顺序组织。第一部分讲数组结构和区间语义，目的是把“下标为什么这样写”说清楚。第二部分讲数组题本身，重点是扫描、最大子数组、前缀和、二维前缀和、差分、除自身以外数组的乘积、移动零和轮转数组。第三部分讲双指针，它不再被当成开场技巧，而是从数组枚举和候选淘汰自然推出。第四部分讲滑动窗口，专门处理连续区间状态能够增删维护的题。这样读下来，双指针和窗口不是孤立招式，而是数组基础能力向复杂题型延伸的结果。

\section{数组结构和下标语义}

数组的第一条性质是连续存储。以 \code{std::vector<int>} 为例，元素在内存中按顺序排列，\code{nums[i]} 可以理解为从起始地址出发偏移 \code{i} 个元素。这个性质带来两个直接结果。第一，下标访问是 \complexity{O(1)}，二分、双指针、DP 表和前缀数组都依赖这一点。第二，顺序扫描通常有很好的缓存局部性，硬件会倾向于预取相邻内存，所以一个简单的 \code{for} 循环往往比看起来“复杂度相同”的链表遍历更快。

连续存储也有代价。数组中间插入或删除元素时，后面的元素要整体移动；\code{vector} 尾部追加虽然摊还是 \complexity{O(1)}，但容量不足时会重新申请更大的内存并搬迁已有元素，旧指针、引用和迭代器可能失效。刷题里如果题目要求频繁在中间插入删除，数组不一定是合适结构；如果题目主要是读取、扫描、排序、按下标交换或维护区间信息，数组通常是最稳定的起点。

\begin{principlebox}{图示：vector 的物理模型}
\begin{tabular}{@{}p{0.20\linewidth}p{0.72\linewidth}@{}}
对象本身 & 保存元素区起点、当前结束位置、容量结束位置这一类元信息。\\
元素区域 & 一段连续内存，\code{data() + i} 指向第 \code{i} 个元素。\\
\code{size()} & 已经构造的元素个数，可以安全读取的范围是 \code{[0, size())}。\\
\code{capacity()} & 当前内存最多能容纳的元素数量，不等于可以下标访问的元素数量。\\
扩容后果 & 元素被搬到新区域，原来的指针、引用和迭代器不能继续使用。\\
\end{tabular}
\end{principlebox}

很多数组题的错误不是算法选择错，而是区间语义混乱。闭区间 \code{[left, right]} 包含两端，长度是 \code{right - left + 1}；半开区间 \code{[left, right)} 包含左端、不包含右端，长度是 \code{right - left}。前缀和天然适合半开区间，因为 \code{prefix[i]} 表示前 \code{i} 个元素，区间 \code{[left, right)} 的和就是 \code{prefix[right] - prefix[left]}。而很多窗口代码写成闭区间，因为右端下标 \code{right} 每次加入当前元素后立刻更新答案。两种写法都可以，但一段代码内部必须统一。

如果把区间写清楚，很多边界就不需要猜。例如空区间在半开语义下可以表示为 \code{[i, i)}，长度自然为 0；前缀数组多开一个第 0 位后，\code{left == 0} 不再需要特殊分支；窗口收缩时先移出 \code{s[left]} 再 \code{++left}，还是先 \code{++left} 再移出旧字符，必须和窗口定义一致。数组题的下标不是装饰，它就是算法不变量的外在形式。

数组题还要先判断候选对象是什么。候选对象通常只有几类：单个元素、下标对、连续区间、若干个分割点、排列或子集。单个元素问题多半是扫描、计数或比较；下标对问题可能走向哈希表、排序双指针或二分；连续区间问题可能走向前缀和、差分、滑动窗口、单调队列；分割点问题经常走向动态规划或贪心。候选对象没分清，就会把所有题都看成模板题。

复杂度分析也要从候选空间开始。枚举每个元素是 \complexity{O(n)}，枚举每个下标对是 \complexity{O(n^2)}，枚举每个连续子数组也是 \complexity{O(n^2)}，如果每个区间还要重新求和，就会变成 \complexity{O(n^3)}。优化并不是凭空少一层循环，而是保存了暴力过程中反复计算的信息。前缀和保存历史总和，差分保存区间变化端点，前后缀保存左右两侧贡献，双指针利用有序性或瓶颈关系淘汰候选，窗口保存当前连续区间状态。

\section{数组题：扫描、前缀、差分和原地写入}

先看不需要双指针的数组题。它们训练的是数组基本功：如何扫描，如何保存历史，如何把区间信息变成端点信息，如何在原数组上写回结果。这些能力没有练熟，后面写双指针和窗口时很容易只记住移动方向，却忘记状态到底表示什么。

最大子数组和是数组扫描的核心样本。题目要求找一个连续子数组，使元素和最大。最直接的暴力做法是枚举左端和右端，再计算区间和；如果每个区间都重新累加，复杂度会到 \complexity{O(n^3)}。稍微改进后，可以固定左端，右端向右扩展时累加当前和，这样复杂度降到 \complexity{O(n^2)}。这两个版本都正确，因为它们覆盖了所有连续子数组；慢点在于同一段历史贡献被反复使用。

\begin{lstlisting}[language=C++,caption={LeetCode 53 最大子数组和：从暴力区间扩展开始}]
int max_subarray_quadratic(const std::vector<int>& nums)
{
    int best = std::numeric_limits<int>::min();
    for (int left = 0; left < static_cast<int>(nums.size()); ++left) {
        int sum = 0;
        for (int right = left; right < static_cast<int>(nums.size()); ++right) {
            sum += nums[right];
            best = std::max(best, sum);
        }
    }
    return best;
}
\end{lstlisting}

进一步优化时，不要直接背“Kadane 算法”。先问：如果一个以前缀结尾的连续段已经是负数，它接在当前元素前面会让当前元素变差，应该丢弃；如果以前一个位置结尾的最佳连续段是正数，它接上当前元素会更好，应该保留。因此只需要维护一个状态：\code{ending_here} 表示必须以当前位置结尾的最大子数组和。答案 \code{best} 是扫描过程中所有 \code{ending_here} 的最大值。

\begin{lstlisting}[language=C++,caption={LeetCode 53 最大子数组和：线性扫描}]
int max_subarray_linear(const std::vector<int>& nums)
{
    int ending_here = nums.front();
    int best = nums.front();

    for (int i = 1; i < static_cast<int>(nums.size()); ++i) {
        ending_here = std::max(nums[i], ending_here + nums[i]);
        best = std::max(best, ending_here);
    }
    return best;
}
\end{lstlisting}

这段代码的关键不是公式，而是不变量。\code{ending_here} 只负责“必须以当前下标结尾”的最优答案，所以它可以只从两个选择里来：要么当前元素单独开头，要么接在上一个位置的最优后缀后面。它不是全局答案；全局答案由 \code{best} 保存。把这两个含义分开，才能自然处理全负数组。如果把 \code{ending_here} 初始化成 0，就可能错误地返回空子数组的和。

前缀和是数组题最基础的缓存。它适合静态数组上的区间求和、区间计数和差值判断。令 \code{prefix[i]} 表示前 \code{i} 个元素之和，那么闭区间 \code{[left, right]} 的和等于 \code{prefix[right + 1] - prefix[left]}。多出来的第 0 位是哨兵，它让 \code{left == 0} 的情况也服从同一条公式。

\begin{lstlisting}[language=C++,caption={一维前缀和：区间和查询}]
std::vector<std::int64_t> build_prefix_sum(const std::vector<int>& nums)
{
    std::vector<std::int64_t> prefix(nums.size() + 1, 0);
    for (std::size_t i = 0; i < nums.size(); ++i) {
        prefix[i + 1] = prefix[i] + nums[i];
    }
    return prefix;
}

std::int64_t range_sum_closed(const std::vector<std::int64_t>& prefix,
                           int left,
                           int right)
{
    return prefix[static_cast<std::size_t>(right + 1)] -
           prefix[static_cast<std::size_t>(left)];
}
\end{lstlisting}

前缀和的使用前提是聚合操作能被“两个前缀的差”还原。加法可以，异或也可以；最大值和最小值不行，因为知道前缀最大值无法推出中间区间最大值。若题目问区间最大最小，应该考虑单调队列、稀疏表、线段树或堆，而不是硬套前缀和。若数组会频繁单点修改，普通前缀和也不适合，因为一次修改会影响后面所有前缀；这时要考虑树状数组或线段树。

二维前缀和把矩形区域求和变成四个角的组合。它的本质是容斥：目标矩形等于左上到右下的大矩形，减去上方和左方，再加回被减了两次的左上角。实现时建议多开一行一列哨兵，避免在第一行、第一列写特殊分支。

\begin{lstlisting}[language=C++,caption={二维前缀和：矩形查询}]
class MatrixPrefixSum {
public:
    explicit MatrixPrefixSum(const std::vector<std::vector<int>>& matrix)
    {
        const int rows = static_cast<int>(matrix.size());
        const int cols = rows == 0 ? 0 : static_cast<int>(matrix[0].size());
        this->prefix_.assign(rows + 1, std::vector<int>(cols + 1, 0));

        for (int row = 0; row < rows; ++row) {
            for (int col = 0; col < cols; ++col) {
                this->prefix_[row + 1][col + 1] =
                    this->prefix_[row][col + 1] +
                    this->prefix_[row + 1][col] -
                    this->prefix_[row][col] +
                    matrix[row][col];
            }
        }
    }

    int query(int top, int left, int bottom, int right) const
    {
        return this->prefix_[bottom + 1][right + 1] -
               this->prefix_[top][right + 1] -
               this->prefix_[bottom + 1][left] +
               this->prefix_[top][left];
    }

private:
    std::vector<std::vector<int>> prefix_;
};
\end{lstlisting}

\code{prefix[row + 1][col + 1]} 表示原矩阵从 \code{(0,0)} 到 \code{(row,col)} 的闭矩形和。查询时 \code{top}、\code{left}、\code{bottom}、\code{right} 都是原矩阵坐标，所以访问前缀数组时右下角要加一。二维前缀和常见错误是把“原矩阵坐标”和“前缀矩阵坐标”混在一起。解决办法是始终记住：前缀矩阵的第 0 行和第 0 列不是原数据，它们只是边界哨兵。

差分数组是前缀和的反向思维。前缀和把多次区间查询变快；差分把多次区间修改变快。若对闭区间 \code{[left, right]} 加上 \code{delta}，不需要逐个位置修改，只需要让变化从 \code{left} 开始，在 \code{right + 1} 结束。所有更新记录完以后，对差分数组做一次前缀累加，就得到最终数组。

\begin{lstlisting}[language=C++,caption={差分数组：区间更新后一次还原}]
std::vector<int> apply_range_additions(
    int length,
    const std::vector<std::tuple<int, int, int>>& updates)
{
    std::vector<int> diff(length + 1, 0);
    for (const auto [left, right, delta] : updates) {
        diff[left] += delta;
        if (right + 1 < length) {
            diff[right + 1] -= delta;
        }
    }

    std::vector<int> values(length, 0);
    int current = 0;
    for (int i = 0; i < length; ++i) {
        current += diff[i];
        values[i] = current;
    }
    return values;
}
\end{lstlisting}

差分的暴力基线是：每次区间更新都扫描区间内每个位置，若有 \code{q} 次更新、数组长度为 \code{n}，最坏是 \complexity{O(qn)}。差分优化后，每次更新只改两个端点，总共 \complexity{O(q)}，最后还原一次 \complexity{O(n)}。它适合离线批量更新。如果每次更新后都要立刻查询某个区间，普通差分就不够，需要树状数组、线段树或扫描线结构。

除自身以外数组的乘积展示前后缀思想。题目要求不使用除法，返回 \code{answer[i]} 等于除 \code{nums[i]} 以外所有元素的乘积。暴力做法是对每个 \code{i} 再扫描整个数组，跳过自己，复杂度 \complexity{O(n^2)}。慢点在于左侧乘积和右侧乘积被反复计算。优化时，把答案拆成两部分：\code{i} 左侧所有数的乘积乘以 \code{i} 右侧所有数的乘积。

\begin{lstlisting}[language=C++,caption={LeetCode 238 除自身以外数组的乘积：前后缀合并}]
std::vector<int> product_except_self(const std::vector<int>& nums)
{
    std::vector<int> answer(nums.size(), 1);

    int left_product = 1;
    for (std::size_t i = 0; i < nums.size(); ++i) {
        answer[i] = left_product;
        left_product *= nums[i];
    }

    int right_product = 1;
    for (std::size_t i = nums.size(); i-- > 0;) {
        answer[i] *= right_product;
        right_product *= nums[i];
    }

    return answer;
}
\end{lstlisting}

第一遍扫描时，\code{answer[i]} 先保存 \code{i} 左侧乘积；第二遍从右向左扫描，用 \code{right_product} 乘上右侧贡献。这里输出数组本身被当作状态存储，所以额外空间可以视为 \complexity{O(1)}，不计返回值。零值不需要特殊分支，因为前后缀乘积自然处理了一个零或多个零的情况。若题目数据范围可能溢出，应主动说明需要更大整数类型或按题目保证处理。

原地数组题训练的是写入位置和读取位置分离。移动零要求把所有非零元素按原相对顺序移到前面，零放到后面。暴力思路可以新建一个数组，先收集非零，再补零；如果题目要求原地，就要把“结果数组的写入位置”映射回原数组。慢指针 \code{write} 表示下一个非零元素应该写入的位置，快指针 \code{read} 扫描原输入。

\begin{lstlisting}[language=C++,caption={LeetCode 283 移动零：稳定覆盖写入}]
void move_zeroes(std::vector<int>& nums)
{
    int write = 0;
    for (int read = 0; read < static_cast<int>(nums.size()); ++read) {
        if (nums[read] != 0) {
            nums[write] = nums[read];
            ++write;
        }
    }

    while (write < static_cast<int>(nums.size())) {
        nums[write] = 0;
        ++write;
    }
}
\end{lstlisting}

这段代码的不变量是：\code{[0, write)} 已经保存了扫描过元素中的全部非零值，且相对顺序不变；\code{read} 之前的元素已经处理完；\code{read} 之后仍未知。注意这不是本章后面要讲的“候选淘汰型双指针”，而是“读写分离型数组原地维护”。它也解释了删除有序数组重复项、移除元素、稳定过滤等题的共同结构：慢指针不是在找答案边界，而是在维护已整理前缀。

轮转数组展示另一类原地操作：整体搬运。题目要求把数组右移 \code{k} 位。新建数组很容易，\codeesc{answer[(i + k) \% n] = nums[i]}，但若要求原地，就不能依赖额外数组。三次反转的思路是把数组分成前 \code{n-k} 个元素和后 \code{k} 个元素，右移后后段应来到前面、前段应来到后面。整体反转会改变两段顺序，分别再反转两段即可恢复各段内部顺序。

\begin{lstlisting}[language=C++,caption={LeetCode 189 轮转数组：三次反转}]
void rotate_array(std::vector<int>& nums, int k)
{
    if (nums.empty()) {
        return;
    }

    k %= static_cast<int>(nums.size());
    std::reverse(nums.begin(), nums.end());
    std::reverse(nums.begin(), nums.begin() + k);
    std::reverse(nums.begin() + k, nums.end());
}
\end{lstlisting}

三次反转是很适合面试的版本，因为不变量清晰、边界少、复杂度稳定。环状替换也能做到原地，但要处理循环个数和最大公约数，解释成本更高。这里要注意 \code{k} 可能大于数组长度，所以必须先取模；数组为空时不能取模，否则会除以零。

\begin{principlebox}{数组题复盘模板}
数组题复盘时先写暴力枚举的候选空间，再指出慢点是哪一种重复工作：重复求区间和、重复计算左右贡献、重复搬移元素、重复统计窗口频次，还是重复检查已经被支配的候选。之后再说明保存了什么状态，以及这个状态如何随下标推进。只写“用前缀和”“用差分”“用原地双指针”还不够，必须把状态含义和边界语义讲清楚。
\end{principlebox}

\section{双指针：从数组枚举到候选淘汰}

讲完数组本体和数组题，再看双指针就自然很多。双指针不是“用了两个变量”这么简单。它至少有三类常见模型：两端收缩、排序配对、读写分离。上一节的移动零属于读写分离，慢指针维护已整理前缀；这一节重点讲两端收缩和排序配对，因为它们真正依赖候选淘汰。

双指针能成立，必须给出移动指针的理由。两端收缩题里，移动某一侧意味着排除一批候选，必须证明这些候选不可能成为更优答案；排序配对题里，和太小移动左端、和太大移动右端，依赖数组有序；如果数组无序，就没有单调性可以使用。很多题解只说“左指针右移、右指针左移”，这不算理解。正确表达应该是：当前状态排除了哪些候选，为什么它们一定不需要再看。

盛最多水的容器是两端收缩的经典题。暴力解枚举左右边界，每个下标对都可能是候选。面积由宽度和较短边共同决定，所以暴力复杂度是 \complexity{O(n^2)}。

\begin{lstlisting}[language=C++,caption={LeetCode 11 盛最多水的容器：暴力解}]
int max_area_bruteforce(const std::vector<int>& height)
{
    int best = 0;
    for (int left = 0; left < static_cast<int>(height.size()); ++left) {
        for (int right = left + 1; right < static_cast<int>(height.size()); ++right) {
            const int width = right - left;
            const int bounded_height = std::min(height[left], height[right]);
            best = std::max(best, width * bounded_height);
        }
    }
    return best;
}
\end{lstlisting}

优化从最宽的左右边界开始。若当前 \code{height[left] <= height[right]}，面积瓶颈是左边。此时如果移动右边，宽度变小，短板高度仍然不超过原来的左边高度，所以不可能得到更大面积。唯一有机会变好的方向是移动左边，期待新的左边更高。右边更短时同理移动右边。

\begin{lstlisting}[language=C++,caption={LeetCode 11 盛最多水的容器：双指针解}]
int max_area_two_pointers(const std::vector<int>& height)
{
    int left = 0;
    int right = static_cast<int>(height.size()) - 1;
    int best = 0;

    while (left < right) {
        const int width = right - left;
        const int bounded_height = std::min(height[left], height[right]);
        best = std::max(best, width * bounded_height);

        if (height[left] <= height[right]) {
            ++left;
        } else {
            --right;
        }
    }
    return best;
}
\end{lstlisting}

这个证明的核心是“短板支配”。移动长板不会提高短板，只会缩小宽度；移动短板虽然也缩小宽度，但至少可能提高短板。每一步排除的是当前短板和更内侧任意另一边组成的候选，因为它们宽度更小且瓶颈不高于当前短板。把这个排除理由说清楚，比记住代码更重要。

两数之和 II 是排序配对模型。题目给出有序数组，要求返回两数之和等于目标的下标。暴力枚举下标对是 \complexity{O(n^2)}。有序性提供了淘汰依据：当前和太小，固定右端时，左端再往左只会更小，所以左端必须右移；当前和太大，固定左端时，右端再往右只会更大，所以右端必须左移。

\begin{lstlisting}[language=C++,caption={LeetCode 167 两数之和 II：有序数组配对搜索}]
std::vector<int> two_sum_sorted(const std::vector<int>& numbers, int target)
{
    int left = 0;
    int right = static_cast<int>(numbers.size()) - 1;
    while (left < right) {
        const int sum = numbers[left] + numbers[right];
        if (sum == target) {
            return {left + 1, right + 1};
        }
        if (sum < target) {
            ++left;
        } else {
            --right;
        }
    }
    return {};
}
\end{lstlisting}

如果数组无序，这段推理就不存在。无序两数之和通常用哈希表，因为要快速查历史补数，而不是从两端淘汰候选。算法选择必须服务于题目条件：有序数组给双指针，历史等值查询给哈希表，不能因为题名都叫两数之和就套同一段代码。

三数之和把排序配对再推进一步。题目要求找出所有不重复三元组，使三个数之和为 0。暴力方法枚举三个下标是 \complexity{O(n^3)}，还要处理去重。先排序后，固定第一个数，剩余部分就变成有序数组上的两数之和。排序还让去重变得可控：同一层循环中，当前值和前一个值相同就跳过；找到一个答案后，左右指针都跳过连续重复值。

\begin{lstlisting}[language=C++,caption={LeetCode 15 三数之和：排序加双指针}]
std::vector<std::vector<int>> three_sum(std::vector<int> nums)
{
    std::sort(nums.begin(), nums.end());
    std::vector<std::vector<int>> answer;

    for (int first = 0; first < static_cast<int>(nums.size()); ++first) {
        if (first > 0 && nums[first] == nums[first - 1]) {
            continue;
        }
        if (nums[first] > 0) {
            break;
        }

        int left = first + 1;
        int right = static_cast<int>(nums.size()) - 1;
        while (left < right) {
            const int sum = nums[first] + nums[left] + nums[right];
            if (sum == 0) {
                answer.push_back({nums[first], nums[left], nums[right]});
                const int left_value = nums[left];
                const int right_value = nums[right];
                while (left < right && nums[left] == left_value) {
                    ++left;
                }
                while (left < right && nums[right] == right_value) {
                    --right;
                }
            } else if (sum < 0) {
                ++left;
            } else {
                --right;
            }
        }
    }

    return answer;
}
\end{lstlisting}

这里参数按值传入，是因为函数内部要排序，又不想修改调用者的数组。如果题目允许原地排序，也可以传引用。复杂度是排序 \complexity{O(n\log n)} 加双指针枚举 \complexity{O(n^2)}，总时间 \complexity{O(n^2)}，不计答案的额外空间通常视为 \complexity{O(1)} 或排序栈空间。去重不要最后把结果塞进 \code{set}，那会掩盖生成过程中的重复，也让复杂度和常数变差；更好的做法是在每一层候选生成时就避免重复。

四数之和是三数之和的自然扩展。暴力四层循环是 \complexity{O(n^4)}。排序后固定前两个数，后两个数用双指针，复杂度降到 \complexity{O(n^3)}。去重必须发生在每一层：第一个数重复跳过，第二个数重复跳过，找到答案后左右指针跳过重复。还要注意整数溢出，四个 \code{int} 相加可能超过 \code{int} 范围，应该用 \code{std::int64_t} 保存和。

\begin{lstlisting}[language=C++,caption={LeetCode 18 四数之和：排序、分层去重和长整型求和}]
std::vector<std::vector<int>> four_sum(std::vector<int> nums, int target)
{
    std::sort(nums.begin(), nums.end());
    std::vector<std::vector<int>> answer;
    const int n = static_cast<int>(nums.size());

    for (int first = 0; first < n; ++first) {
        if (first > 0 && nums[first] == nums[first - 1]) {
            continue;
        }
        for (int second = first + 1; second < n; ++second) {
            if (second > first + 1 && nums[second] == nums[second - 1]) {
                continue;
            }

            int left = second + 1;
            int right = n - 1;
            while (left < right) {
                const std::int64_t sum =
                    static_cast<std::int64_t>(nums[first]) + nums[second] +
                    nums[left] + nums[right];
                if (sum == target) {
                    answer.push_back(
                        {nums[first], nums[second], nums[left], nums[right]});
                    const int left_value = nums[left];
                    const int right_value = nums[right];
                    while (left < right && nums[left] == left_value) {
                        ++left;
                    }
                    while (left < right && nums[right] == right_value) {
                        --right;
                    }
                } else if (sum < target) {
                    ++left;
                } else {
                    --right;
                }
            }
        }
    }

    return answer;
}
\end{lstlisting}

四数之和可以继续加剪枝，例如当前最小可能和已经大于目标就退出，当前最大可能和仍小于目标就跳过。但剪枝必须处理好溢出和重复值，否则容易为了减少几次循环破坏正确性。教材阶段更建议先写稳定版本，再说明可选剪枝。面试中表达重点也不是“我会写四层结构”，而是“排序给了后两数单调性，去重在每层生成候选时完成，求和用长整型避免溢出”。

\begin{principlebox}{双指针自查}
使用双指针前先问三件事。第一，两个指针分别代表什么候选边界或读写位置。第二，移动某个指针时排除了哪些候选。第三，被排除的候选为什么不可能成为答案。若第三点说不出来，就不要把它称为双指针优化。
\end{principlebox}

\section{滑动窗口：连续区间的增量状态}

滑动窗口建立在数组连续区间之上。它不是所有“有 left 和 right 的代码”，而是要求三个条件同时成立：答案对象是连续区间；边界移动后状态能用常数或近似常数成本增删；移动策略不会漏掉答案。正整数数组的最短子数组和可以用窗口，因为右端扩展只会让和变大，左端收缩只会让和变小。含负数数组就不能直接这样做，因为负数进入可能让和变小，负数离开可能让和变大，窗口判断失去单调性。

窗口题先分固定长度和可变长度。固定长度窗口的长度由题目直接给出，例如找到所有异位词、长度为 K 的最大平均值、滑动窗口最大值。可变长度窗口的长度由约束决定，例如无重复字符、最小覆盖子串、乘积小于 K 的子数组。固定窗口通常每步右进左出；可变窗口通常右端扩展到满足或破坏条件，再移动左端恢复条件或压缩答案。

长度最小的子数组是最适合入门的可变窗口题。题目给定正整数数组和目标值，要求找和至少为目标的最短连续子数组。暴力法枚举所有起点和终点，遇到和达到目标时更新答案。它慢在每个起点都重新累加大量重叠区间。由于元素全为正数，右端扩展会让和增加，左端收缩会让和减少，所以可以维护一个窗口和。

\begin{lstlisting}[language=C++,caption={LeetCode 209 长度最小的子数组：滑动窗口}]
int min_subarray_len_sliding_window(int target, const std::vector<int>& nums)
{
    int best = std::numeric_limits<int>::max();
    int left = 0;
    int sum = 0;

    for (int right = 0; right < static_cast<int>(nums.size()); ++right) {
        sum += nums[right];
        while (sum >= target) {
            best = std::min(best, right - left + 1);
            sum -= nums[left];
            ++left;
        }
    }

    return best == std::numeric_limits<int>::max() ? 0 : best;
}
\end{lstlisting}

这里的窗口不变量是：外层循环把 \code{nums[right]} 加入后，窗口为闭区间 \code{[left, right]}；内层循环在窗口已经满足目标时尽可能收缩左端，并在每次收缩前记录一个合法答案。每个元素最多被右端加入一次、被左端移出一次，所以复杂度是 \complexity{O(n)}。如果题目改成允许负数，这个推理立刻失效，不能继续使用普通窗口。

无重复字符的最长子串也是可变窗口，但状态不是求和，而是字符最近出现位置。窗口不变量是 \code{[left, right]} 内没有重复字符。处理新字符 \code{s[right]} 时，如果它上次出现的位置在当前窗口内，左边界至少要跳到上次位置后一位；如果上次出现位置在窗口左侧之外，左边界不能回退。

\begin{lstlisting}[language=C++,caption={LeetCode 3 无重复字符的最长子串}]
int longest_substring_without_repeat_window(const std::string& s)
{
    constexpr int ASCII_RANGE = 256;
    std::array<int, ASCII_RANGE> last_seen{};
    last_seen.fill(-1);

    int left = 0;
    int best = 0;

    for (int right = 0; right < static_cast<int>(s.size()); ++right) {
        const auto ch = static_cast<unsigned char>(s[right]);
        if (last_seen[ch] >= left) {
            left = last_seen[ch] + 1;
        }
        last_seen[ch] = right;
        best = std::max(best, right - left + 1);
    }

    return best;
}
\end{lstlisting}

这题的关键错误是让 \code{left} 回退。输入 \code{abba} 时，处理最后一个 \code{a}，它上次出现位置是 0，但当前窗口左边界已经在 2；如果直接写 \code{left = last_seen[ch] + 1}，就会把左边界错误地拉回 1。正确条件是只有当上次出现位置仍在当前窗口内，才更新左边界。字符集如果不是 ASCII，就要换成哈希表或先明确编码处理方式。

找到字符串中所有字母异位词是固定长度窗口。题目给定字符串 \code{s} 和模式串 \code{p}，要求找出 \code{s} 中所有长度等于 \code{p} 且字符频次完全相同的起点。暴力方法枚举每个长度为 \code{p.size()} 的子串，再排序比较或统计比较。如果每次排序，复杂度是 \complexity{O(nk\log k)}；如果每次重新统计，复杂度是 \complexity{O(nk)}。优化点是相邻窗口只有一个字符离开、一个字符进入。

\begin{lstlisting}[language=C++,caption={LeetCode 438 找到所有字母异位词：固定长度窗口}]
std::vector<int> find_anagrams_window(const std::string& s,
                                      const std::string& p)
{
    constexpr int ALPHABET_SIZE = 26;
    std::vector<int> answer;
    if (s.size() < p.size()) {
        return answer;
    }

    std::array<int, ALPHABET_SIZE> need{};
    std::array<int, ALPHABET_SIZE> window{};
    for (const char ch : p) {
        ++need[ch - 'a'];
    }

    for (std::size_t right = 0; right < s.size(); ++right) {
        ++window[s[right] - 'a'];
        if (right >= p.size()) {
            --window[s[right - p.size()] - 'a'];
        }
        if (window == need) {
            answer.push_back(static_cast<int>(right + 1 - p.size()));
        }
    }

    return answer;
}
\end{lstlisting}

窗口第一次达到目标长度是在 \code{right + 1 == p.size()}，起点是 \code{right + 1 - p.size()}。如果用 \code{int} 写下标，要小心 \code{s.size()} 是无符号类型；这里使用 \code{std::size_t} 控制循环，最终起点再转成 \code{int}，因为力扣返回 \code{vector<int>}。若字符集固定为小写英文，比较两个长度为 26 的数组可以视为常数；若字符集很大，可以维护差异计数，避免每步全量比较。

最小覆盖子串是变长窗口里最重要的一题。题目要求在 \code{s} 中找最短子串，覆盖 \code{t} 中所有字符及其次数。暴力方法枚举所有子串，再检查是否覆盖，复杂度至少 \complexity{O(n^2 |\Sigma|)}。优化的核心是：右边界扩张直到窗口满足覆盖条件，然后尽可能移动左边界缩短窗口；一旦左移导致窗口不满足，再继续右移。

这题最容易被讲成“用滑动窗口”，但读者需要看到窗口是怎样从暴力里长出来的。以 \code{s = "ADOBECODEBANC"}、\code{t = "ABC"} 为例，暴力版会先固定左端 0，再让右端从 0 扫到 5，第一次得到 \code{"ADOBEC"}，它覆盖 A、B、C；继续右移虽然仍覆盖，但窗口只会更长，对左端 0 来说已经没有必要再作为最短答案。于是我们自然想把左端右移。左端从 0 移走字符 A 后，窗口失去 A，不再覆盖目标，只能继续让右端右移寻找新的 A。这个过程说明了两个状态：窗口内每个字符的计数，以及已经满足需求的字符种类数。它们取代了暴力版中“每次重新检查整个子串”的工作。

扩张 & \code{[0,5]} & \code{ADOBEC} & 第一次覆盖 A、B、C，记录长度 6。\\

从这张表可以推出代码里的 \code{required} 和 \code{formed}。如果每次判断覆盖都比较 128 个字符计数，也能做，但状态表达不够聚焦。 \code{required} 表示目标里有多少种不同字符需要满足；\code{formed} 表示当前窗口里已经有多少种字符的计数达到目标。某个字符计数从不足变成刚好满足时，\code{formed} 加一；从满足变回不足时，\code{formed} 减一。窗口合法条件就从“全表比较”变成 \code{formed == required}。

\begin{lstlisting}[language=C++,caption={LeetCode 76 最小覆盖子串：变长窗口}]
std::string min_window_substring(const std::string& s, const std::string& t)
{
    constexpr int ASCII_RANGE = 128;
    std::array<int, ASCII_RANGE> need{};
    std::array<int, ASCII_RANGE> window{};
    int required = 0;

    for (const char ch : t) {
        const auto index = static_cast<unsigned char>(ch);
        if (need[index] == 0) {
            ++required;
        }
        ++need[index];
    }

    int formed = 0;
    int left = 0;
    int best_left = 0;
    int best_length = std::numeric_limits<int>::max();

    for (int right = 0; right < static_cast<int>(s.size()); ++right) {
        const auto in = static_cast<unsigned char>(s[right]);
        ++window[in];
        if (need[in] > 0 && window[in] == need[in]) {
            ++formed;
        }

        while (formed == required) {
            const int length = right - left + 1;
            if (length < best_length) {
                best_length = length;
                best_left = left;
            }

            const auto out = static_cast<unsigned char>(s[left]);
            --window[out];
            if (need[out] > 0 && window[out] < need[out]) {
                --formed;
            }
            ++left;
        }
    }

    if (best_length == std::numeric_limits<int>::max()) {
        return "";
    }
    return s.substr(best_left, best_length);
}
\end{lstlisting}

\code{required} 表示需要满足的不同字符种类数，\code{formed} 表示当前窗口里已经满足需求的字符种类数。不能只用窗口总长度判断，因为 \code{t = "AABC"} 时，两个 \code{A} 都必须覆盖。窗口满足时收缩左端，是为了找最短；窗口不满足时扩展右端，是为了重新获得覆盖能力。这个过程每个字符最多进入一次、离开一次，所以是线性扫描。

乘积小于 K 的子数组展示窗口计数。数组元素为正，窗口乘积随着右端扩展会变大，随着左端收缩会变小，因此具有单调性。对每个右端，当窗口乘积小于 \code{k} 时，以右端结尾的合法子数组数量是 \code{right - left + 1}，因为从 \code{left} 到 \code{right} 的任意起点都能得到合法乘积。若 \code{k <= 1}，正整数乘积不可能小于 \code{k}，直接返回 0。

\begin{lstlisting}[language=C++,caption={LeetCode 713 乘积小于 K 的子数组：窗口计数}]
int num_subarray_product_less_than_k(const std::vector<int>& nums, int k)
{
    if (k <= 1) {
        return 0;
    }

    int left = 0;
    int product = 1;
    int answer = 0;
    for (int right = 0; right < static_cast<int>(nums.size()); ++right) {
        product *= nums[right];
        while (product >= k) {
            product /= nums[left];
            ++left;
        }
        answer += right - left + 1;
    }
    return answer;
}
\end{lstlisting}

这题经常被错写成只更新最长长度，但题目问的是子数组数量。窗口恢复合法后，所有以 \code{right} 结尾、起点在 \code{[left, right]} 的子数组都合法，因此一次加上窗口长度。计数题要特别注意“每个答案由哪个时刻统计”。这里每个合法子数组按右端点唯一统计一次，所以不会重复。

有些看起来像窗口的题不能用普通滑动窗口，但可以转成前缀和加单调队列。最短子数组和至少为 K 允许负数，窗口和不再随右端扩展单调增加。改用前缀和后，问题变成找 \code{prefix[right] - prefix[left] >= K} 且 \code{right - left} 最小。扫描右端时，用单调队列维护可能的左端前缀下标，并让前缀和递增。

\begin{lstlisting}[language=C++,caption={LeetCode 862 和至少为 K 的最短子数组：前缀和加单调队列}]
int shortest_subarray_at_least_k(const std::vector<int>& nums, int k)
{
    const int n = static_cast<int>(nums.size());
    std::vector<std::int64_t> prefix(n + 1, 0);
    for (int i = 0; i < n; ++i) {
        prefix[i + 1] = prefix[i] + nums[i];
    }

    std::deque<int> deque;
    int answer = n + 1;
    for (int right = 0; right <= n; ++right) {
        while (!deque.empty() &&
               prefix[right] - prefix[deque.front()] >= k) {
            answer = std::min(answer, right - deque.front());
            deque.pop_front();
        }
        while (!deque.empty() && prefix[deque.back()] >= prefix[right]) {
            deque.pop_back();
        }
        deque.push_back(right);
    }

    return answer == n + 1 ? -1 : answer;
}
\end{lstlisting}

为什么队尾前缀和更大时可以删除？若有两个候选左端 \code{i < j} 且 \code{prefix[i] >= prefix[j]}，那么对于任何未来右端 \code{r}，\code{prefix[r] - prefix[j]} 不小于 \code{prefix[r] - prefix[i]}，并且 \code{j} 更靠右，区间更短。因此 \code{i} 被 \code{j} 完全支配，可以删除。这题放在窗口章节末尾，是为了提醒：窗口思维可以启发我们维护连续区间，但题目条件一变，普通窗口就要升级成前缀和与候选队列。

\begin{warningbox}
滑动窗口有三个常见误用。第一，数组含负数却用窗口维护和的单调性。第二，题目要求非连续子序列，却套连续窗口。第三，窗口状态不能常数时间增删，却假设移动边界很便宜。使用窗口前必须确认：答案是连续区间，边界移动后状态能增量维护，且移动策略不会漏掉可能答案。
\end{warningbox}
