# 第 0 章：序言、目标和学习方法

刷题最容易走偏的方式，是把每一道题当成孤立技巧。今天背一个双指针，明天背一个单调栈，后天背一个动态规划转移式，看起来题量很多，面试时却仍然卡在“我怎么想到这个解法”。本书要解决的就是这个问题：不只告诉你答案是什么，还要训练你从暴力方法一步步走到优化方法。

## 本书的目标

三个月结束时，你应该达到几个具体标准。

第一，看到一道中等难度题，能先写出正确暴力解或半暴力解。暴力解不是低级阶段，它是理解题目的第一张地图。不会暴力，通常说明你还没搞清楚枚举对象、状态含义和正确性边界。

第二，能解释优化从哪里来。优化不是玄学，常见来源只有几类：枚举里有重复计算，可以缓存；搜索空间有单调性，可以二分或双指针；局部状态足够表达全局约束，可以滑动窗口；需要快速查询某种历史信息，可以哈希、堆、单调队列或树状数组；问题可以拆成重叠子问题，可以动态规划。

第三，能用 C++ 写出稳定代码。面试代码不要求工程项目级封装，但必须边界清楚、复杂度合理、容器选择正确。比如 `vector` 适合连续存储和下标访问，`deque` 适合两端操作，`unordered_map` 适合平均 O(1) 查询但有哈希退化和 rehash 成本，`priority_queue` 只能访问堆顶，不能随意删除中间元素。

第四，能复盘题型。刷题不是做完提交 AC 就结束，而是要回答：这题的暴力解是什么？优化靠什么性质？这个性质在什么条件下失效？换成负数、重复元素、在线数据流、返回方案而不是返回数量，解法是否还成立？

## 一道题的标准处理流程

面对一道算法题，按下面顺序走。

读题时先标记四件事：输入规模、值域、是否有序、是否允许重复。输入规模决定复杂度上限，值域决定能不能用数组计数或位图，有序性决定能不能双指针或二分，重复元素决定哈希表和去重逻辑是否需要特殊处理。

然后写暴力解。暴力解要讲清楚枚举什么。例如两数之和枚举两个下标，最长无重复子串枚举所有起点和终点，岛屿数量枚举每个格子再扩展连通块。暴力解的价值在于它把问题的“解空间”显式摆出来。

接着找重复工作。两数之和里，内层循环反复寻找 `target - nums[i]`，这就是哈希表能优化的原因。最长无重复子串里，从每个起点重新建集合，会反复扫描同一段字符，这就是滑动窗口能优化的原因。腐烂橘子里，每分钟所有腐烂橘子同时扩散，这就是多源 BFS 的原因。

最后写优化解，并给出不变量。不变量是面试表达的核心。滑动窗口的不变量可能是“窗口内没有重复字符”；双指针盛水的不变量是“每次移动较短边不会错过最优解”；BFS 的不变量是“队列里当前层的所有节点距离起点相同”。

## 每天怎么学

每天至少保留三段时间。第一段读一个主题，整理模板和关键性质。第二段做题，先自己写暴力，再看能不能优化。第三段复盘，把错因写下来。错因不要写“没想到”，要写具体：没看出单调性、窗口收缩条件错、哈希表更新顺序错、二分边界写错、`long long` 没用导致溢出。

## 本书代码风格

示例代码使用 C++20。函数参数优先使用 `const std::vector<int>&`、`const std::string&` 这类只读引用，避免不必要拷贝。需要修改输入来标记访问状态时，函数会按值接收副本，例如网格 BFS 示例用 `std::vector<std::vector<int>> grid`，这样既能展示原地标记，也不会修改调用者数据。

本书尽量使用迭代写法。递归 DFS 在刷题中常见，但 C++ 面试和线上评测里容易遇到栈深问题；本书会讲递归思想，但实现上会优先展示显式栈、队列和循环。
