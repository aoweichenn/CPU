# Practice And Code Volumes

实践与代码卷把分阶段任务、实验步骤、源码阅读、完整工程清单、测试、benchmark 和报告验收放在同一条目录链路里。

当前入口：

- `cpu_volume_1_practice` -> `../../cpu-volume-1-practice`
- `cpu_volume_3_practice_code` -> `../../cpu-volume-3-practice`
- `compute_systems_engine_code` -> `../../compute-systems-engine-code`

后续计算机组成原理、操作系统和网络都应该优先采用这类合并结构：同一本配套卷先给任务和验收，再给源码阅读路径和完整代码证据，避免实践卷和代码卷互相跳转。
