\chapter*{术语表}

\begin{description}
\item[算术强度] 操作数与数据移动字节数的比值，用来判断一个内核更可能受计算吞吐还是内存带宽限制。
\item[Roofline] 用计算峰值、内存带宽和算术强度估算性能上界的模型。
\item[CPU] Central Processing Unit，中央处理器。
\item[GPU] Graphics Processing Unit，图形处理器或通用并行加速器。
\item[OS] Operating System，操作系统，负责调度线程、管理内存、文件、网络和设备。
\item[AMD] Advanced Micro Devices，常见 CPU/GPU 厂商名。性能报告要记录具体平台，不能只写 x86。
\item[C++20] C++ 标准版本名。本册并发、同步和运行时代码示例默认使用的语言能力边界。
\item[MSVC] Microsoft Visual C++，微软 C++ 工具链。源码阅读标准库实现时，要和 libstdc++、libc++ 区分。
\item[ISA] Instruction Set Architecture，指令集架构，规定软件可见的指令、寄存器和状态。
\item[API] Application Programming Interface，源码层调用合同。
\item[ABI] Application Binary Interface，二进制层调用、对象布局和链接合同。
\item[CFG] Control-Flow Graph，控制流图，把基本块和跳转边连起来。
\item[STL] Standard Template Library，C++ 标准库容器、迭代器和算法的常见统称。
\item[HPC] High-Performance Computing，高性能计算，关注吞吐、并行、带宽和可复现实验。
\item[SIMD] 一条指令处理多个数据 lane 的执行方式，是现代 CPU 单核吞吐的重要来源。
\item[ILP] Instruction-Level Parallelism，指令级并行。互不依赖的指令可以重叠执行，用来隐藏部分延迟。
\item[ASCII] 早期单字节字符集合，适合用 SIMD 做简单字节分类。
\item[UTF-8] Unicode 的常见变长字节编码。解析器要处理跨块 continuation byte 和非法序列。
\item[AVX/AVX2] x86 向量指令扩展，常用于 256 位向量数据并行。
\item[AVX-512] x86 512 位向量扩展，提供更宽寄存器和 mask 机制，但要验证频率、平台覆盖和短输入成本。
\item[ARM] 一类常见处理器架构生态。跨 ARM 和 x86 比较性能时，要重新验证 ISA、cache、内存序和工具事件。
\item[POWER] 一类处理器架构生态，常作为弱内存模型讨论样本。可移植 C++ 同步不能依赖 x86 的偶然强顺序。
\item[RISC] Reduced Instruction Set Computer，精简指令集计算机风格的架构类别。
\item[RISC-V] 开放指令集架构，常作为和 x86、ARM 对照的 ISA 样本。
\item[SSE] Streaming SIMD Extensions，较早的 x86 SIMD 扩展族。
\item[LLVM] 编译器基础设施项目，第二册常用其向量化诊断、IR 和工具链证据。
\item[GCC] GNU 编译器集合，常与 LLVM/Clang 对照观察优化和向量化。
\item[SLP] Superword-Level Parallelism，把相邻相似标量语句合成向量操作的一类向量化方式。
\item[perf] Linux 性能分析工具，可用于采样、读取 PMU 事件、观察调度和调用栈。它提供证据，但需要绑定具体命令、事件和环境。
\item[objdump] 二进制反汇编和目标文件查看工具，常用于确认编译器是否生成了预期指令。
\item[strace] 系统调用跟踪工具，用于确认程序是否频繁进入内核、是否遇到 EINTR/EAGAIN 等状态。
\item[numactl] NUMA 控制和观测工具，可用于绑定 CPU、内存节点或查看 NUMA 拓扑。
\item[VNNI] Vector Neural Network Instructions，面向低精度 dot-product 的向量指令能力。
\item[AMX] Advanced Matrix Extensions，x86 tile 级矩阵扩展，适合足够大的矩阵块计算。
\item[FMA] Fused Multiply-Add，一条指令完成乘加，常用于向量浮点内核。
\item[LTO] Link-Time Optimization，链接期优化，允许跨翻译单元做全局分析和内联。
\item[PGO] Profile-Guided Optimization，基于运行画像反馈优化分支、布局和内联决策。
\item[JIT] Just-In-Time compilation，运行时即时编译或生成代码。
\item[PC] Program Counter，程序计数器，保存下一条取指地址。前端、分支预测、I-cache 和 ITLB 都围绕 PC 流工作。
\item[ALU] Arithmetic Logic Unit，算术逻辑单元，执行整数算术、逻辑和部分比较操作。
\item[AGU] Address Generation Unit，地址生成单元，为 load/store 计算有效地址。
\item[SIB] Scale-Index-Base，x86 地址编码字段，用 base、index、scale 组合有效地址。
\item[LEA] Load Effective Address，x86 地址计算指令，常被编译器用于整数地址/乘加计算。
\item[ROB] Reorder Buffer，重排序缓冲区，保存乱序执行但尚未按程序顺序提交的微操作。
\item[乱序执行] CPU 在保持单线程可观察结果不变的前提下，提前执行已经准备好的指令以隐藏延迟。
\item[分支预测] CPU 在分支结果确定前预测执行路径，预测失败会清空错误路径上的工作。
\item[BTB] Branch Target Buffer，分支目标缓存，用来预测跳转目标地址。
\item[RAS] Return Address Stack，返回地址栈，用来预测函数返回会回到哪里。
\item[cache line] 缓存一致性和缓存填充的基本粒度，常见大小为 64 字节。
\item[MMU] Memory Management Unit，内存管理单元，负责地址翻译和权限检查。
\item[TLB] 地址翻译缓存，用于缓存虚拟页到物理页的映射。
\item[ITLB/DTLB] ITLB 缓存取指地址翻译，DTLB 缓存数据访问地址翻译。二者 miss 的症状和优化手段不同。
\item[PTE] Page Table Entry，页表项，记录虚拟页到物理页的映射和权限。
\item[L1/L2/L3/L1D] CPU 缓存层级。L1D 是一级数据缓存；L3 常是多个核心共享的最后一级缓存。
\item[LLC] Last-Level Cache，最后一级缓存，通常是多个核心共享的较大缓存。
\item[MESI] Modified、Exclusive、Shared、Invalid，常见缓存一致性状态模型，用来解释共享缓存行如何在核心间转移。
\item[MESIF/MOESI] MESI 的一致性协议变体。F 常表示 forward，O 常表示 owned；状态名不同，但共享写要协调所有权这一事实不变。
\item[HITM] Hit Modified，访问的数据命中在其他核心修改过的缓存行，常提示共享写热点或 false sharing。
\item[LL/SC] Load-Linked/Store-Conditional，一类原子更新机制。先建立链接读取，再在链接仍有效时条件写入。
\item[SRAM] Static Random-Access Memory，静态随机访问存储器，常用于 CPU cache。
\item[RFO] Read For Ownership，写入共享缓存行前获取独占所有权的事务。
\item[THP] Transparent Huge Pages，透明大页，系统自动尝试用更大的页减少 TLB 压力。
\item[ULP] Unit in the Last Place，浮点末位单位，用于描述浮点误差尺度。
\item[DRAM] Dynamic Random-Access Memory，主存常见介质，容量大但远慢于缓存。
\item[VM] Virtual Memory，虚拟内存。多线程共享同一进程 VM，所以指针、mmap、堆对象和 data race 都落在同一地址空间。
\item[VMA] Virtual Memory Area，进程地址空间中的连续虚拟内存区域。
\item[VFS] Virtual File System，Linux 中统一文件系统接口的抽象层。
\item[ASLR] Address Space Layout Randomization，地址空间布局随机化，会影响代码、栈、堆和共享库地址。
\item[PMU] Performance Monitoring Unit，性能监控单元，用硬件计数器观察周期、cache miss、分支失败等事件。
\item[TMA] Top-down Microarchitecture Analysis，自顶向下微架构分析方法，把瓶颈粗分为前端受限、后端受限、错误推测和有效退休等类别。
\item[IPC] Instructions Per Cycle，每周期执行指令数。它必须和内存、分支、前后端瓶颈一起解释。
\item[CPI] Cycles Per Instruction，每条指令平均周期数。它和 IPC 互为倒数视角，但解释瓶颈仍要结合事件和代码。
\item[CSV] Comma-Separated Values，逗号分隔文本表格，常用于保存 benchmark 和计数器结果。
\item[GFLOP/s] 每秒十亿次浮点运算，必须说明 FLOP 计数口径和输入形状。
\item[NUMA] 非统一内存访问结构，不同核心访问不同内存节点的成本不同。
\item[SMT] Simultaneous Multithreading，同时多线程，同一物理核心运行多个硬件线程并共享部分资源。
\item[COW] Copy-On-Write，写时复制，先共享对象或页面，写入时再复制。
\item[RMW] Read-Modify-Write，读改写，把读旧值、计算和写回合成一个原子更新序列。
\item[TSO] Total Store Order，x86 常见内存模型名。它比 C++ relaxed/acquire/release 更具体，不能和语言内存序混为一谈。
\item[TSAN] ThreadSanitizer，线程 sanitizer，用来检测数据竞争和部分同步错误。
\item[ASan] AddressSanitizer，发现越界、use-after-free 等内存错误。
\item[UBSan] UndefinedBehaviorSanitizer，发现有符号溢出、非法移位、错误对齐等未定义行为。
\item[sanitizer] 编译器插桩诊断工具族。它们适合正确性检查，不适合直接代表发布性能。
\item[RAII] Resource Acquisition Is Initialization，用 C++ 对象生命周期管理锁、文件、线程句柄和缓冲区等资源。
\item[CV] Condition Variable，条件变量，用于线程等待某个条件变为真。
\item[CFS] Completely Fair Scheduler，Linux 默认调度器族中的公平调度核心思想。
\item[IRIW] Independent Reads of Independent Writes，内存模型 litmus test，用来观察多处理器可见性顺序。
\item[ISR] Interrupt Service Routine，中断服务例程，用来响应硬件或内核事件。
\item[false sharing] 多个线程写不同变量，但变量落在同一 cache line 上，导致一致性流量和扩展性下降。
\item[内存序] 原子操作对可见性和重排施加的约束，例如 relaxed、acquire、release 和 \code{memory_order_seq_cst}。
\item[CAS] Compare-And-Swap，比较并交换，是很多无锁数据结构的基础原子操作。
\item[ABA] 无锁算法中的经典问题：值从 A 变到 B 又变回 A，比较值相同但对象历史已经改变。
\item[lock-free] 一类进度保证，表示系统整体能持续前进，但不保证每个线程有限步完成。
\item[SPSC/MPSC] SPSC 是单生产者单消费者队列，MPSC 是多生产者单消费者队列。
\item[SPMC/MPMC] SPMC 是单生产者多消费者队列，MPMC 是多生产者多消费者队列。
\item[EAGAIN] 常见系统调用错误码，表示资源暂时不可用，调用者应根据阻塞模式和状态机决定是否重试。
\item[EINTR] 系统调用被信号中断。可靠 I/O 循环要明确重试、取消和超时策略。
\item[ETIMEDOUT] 等待超时错误码，常见于 futex、条件变量或带超时的等待路径。
\item[EOF] End Of File，输入结束状态。它不是一个普通字节，读取循环要和短读、错误分开。
\item[POSIX] Portable Operating System Interface，Unix-like 系统常见接口规范。
\item[SIGBUS] Bus error signal，总线错误信号。mmap 文件被截断、映射访问越界或某些对齐/设备访问错误可能触发它。
\item[epoll] Linux I/O 事件通知机制，用文件描述符集合和事件标志驱动非阻塞 I/O。
\item[eventfd/EFD] Linux 事件计数文件描述符及其 flag 前缀，常用于唤醒 event loop 或跨线程通知。
\item[EPOLLIN/EPOLLOUT] epoll 可读和可写事件，分别推动读状态机和写状态机。
\item[EPOLLONESHOT] epoll 一次性触发模式，事件后需要重新 armed。
\item[EPOLLRDHUP] epoll 对端半关闭事件，连接状态机必须把它和普通可读、挂起、错误分开处理。
\item[EPOLLET] epoll 边沿触发模式，只在状态变化时通知。使用它时通常必须循环读/写到 EAGAIN。
\item[EPOLLERR/EPOLLHUP] epoll 错误和挂起事件。可靠连接状态机要显式读取错误、关闭资源并记录诊断。
\item[SO\_ERROR] socket 选项，用于读取异步连接或 I/O 错误原因。
\item[io\_uring] Linux 异步 I/O 接口，用提交队列和完成队列减少系统调用和拷贝成本。
\item[SQ/CQ] io\_uring 的提交队列和完成队列。SQ 表示用户态提交请求的队列，CQ 表示内核写回完成结果的队列。
\item[SQE/CQE] io\_uring 的提交队列项和完成队列项。SQE 表示请求已提交，CQE 表示底层完成有结果。
\item[FIFO] First In First Out，先进先出顺序，常用于队列和管道。
\item[LIFO] Last In First Out，后进先出顺序，典型结构是栈。
\item[PID] Process ID，进程编号，用于定位运行实例和系统资源。
\item[TID] Thread ID，线程编号，用于定位线程调度、亲和性和诊断信息。
\item[RCU] Read-Copy-Update，读路径尽量无锁、更新路径复制新版本并延迟回收旧版本的同步机制。
\item[SLAB/SLUB] Linux 内核小对象分配器族，用对象缓存降低频繁分配释放成本。SLUB 是现代 Linux 常见实现。
\item[TLS] Thread-Local Storage，线程局部存储，每个线程有独立变量实例。
\item[背压] 下游处理不过来时向上游施加的流量控制机制。
\item[DMA/RDMA] DMA 是设备直接访问内存；RDMA 是远端机器通过网卡直接读写内存区域。
\item[UPI] Ultra Path Interconnect，Intel 多 socket 系统中连接 CPU socket 的互连之一，会影响跨 socket 通信和远端内存访问成本。
\item[RPC] Remote Procedure Call，远程过程调用，跨网络和失败边界执行看似本地的调用。
\item[RTT] Round-Trip Time，往返时间，请求从发出到收到响应的端到端网络时间。
\item[TCP] Transmission Control Protocol，可靠字节流协议。它保证有序字节流，不自动提供业务消息边界。
\item[WAL] Write-Ahead Log，预写日志。先持久化意图或状态转移，再发布结果，用于崩溃恢复。
\item[NFS] Network File System，网络文件系统。可靠写、rename 和缓存语义需要按具体环境验证。
\item[DAG] Directed Acyclic Graph，有向无环图，常用于表达任务依赖。
\item[BFS] Breadth-First Search，广度优先搜索。第二册图 kernel 中常用 frontier 表达当前层。
\item[CSR] Compressed Sparse Row，压缩稀疏行格式，常用于图和稀疏矩阵的连续存储。
\item[GEMM] General Matrix-Matrix Multiplication，矩阵乘矩阵，通常具有较强数据复用。
\item[GEMV] General Matrix-Vector Multiplication，矩阵乘向量，常更受权重读流和内存带宽影响。
\item[SLO] Service Level Objective，服务等级目标，例如 p95 延迟或错误率阈值。
\item[P95/P99] 95/99 分位数指标，用来描述尾延迟或尾部资源占用。它比平均值更能暴露排队和抖动。
\item[p50] 50 分位数，也就是中位数。它适合描述典型请求，但不能覆盖尾延迟风险。
\item[RSS] Resident Set Size，常驻内存大小，表示进程当前驻留在物理内存中的页面量。
\item[GC] Garbage Collection，垃圾回收。即便 C++ 主项目不用 GC，分布式系统对比中也会遇到 GC 暂停和内存回收成本。
\item[HDFS] Hadoop Distributed File System，Hadoop 生态的分布式文件系统。
\item[LSM] Log-Structured Merge Tree，日志结构合并树，存储引擎常用结构，写入先进入内存和日志，再分层合并到磁盘文件。
\item[BLIS] 一套高性能 BLAS 风格线性代数库，常用于阅读 blocking、packing、microkernel 和 ISA dispatch 的工程组织。
\item[MR/NR] GEMM microkernel 常见 tile 维度名，通常表示一个内核一次处理的 M/N 方向寄存器块大小。
\item[XNNPACK] 面向神经网络推理的高性能 CPU kernel 库，常用于阅读线程池、数据布局和平台特化 kernel。
\item[LMAX] 常指 LMAX Disruptor 设计案例，用序号环和发布协议组织低延迟队列。
\item[oneTBB] Intel oneAPI Threading Building Blocks，C++ 并行运行时/算法库，常用于对照任务调度、work stealing 和线程池工程边界。
\item[EEVDF] Earliest Eligible Virtual Deadline First，Linux 调度器中的虚拟截止期选择思想，用于解释任务选择和延迟公平性。
\item[futex] Fast userspace mutex，Linux 用户态同步原语和内核等待队列之间的桥。锁快路径留在用户态，竞争时进入内核等待或唤醒。
\item[FUTEX\_WAIT/FUTEX\_WAKE] futex 的等待和唤醒操作。WAIT 检查内存值并睡眠，WAKE 唤醒等待者。
\item[实验常量] 实验或数据结构参数名，例如 \code{SUM_LANE_COUNT}、\code{CACHE_LINE_BYTES}、\code{EVENT_BUCKET_COUNT}。它们用名字固定实验口径，不是新算法。
\item[OOM] Out Of Memory，内存不足。系统设计中要明确谁回收资源、失败是否可恢复。
\item[CSV/JSON] CSV 适合保存表格实验数据；JSON 适合保存结构化配置、manifest 或报告。
\item[DWORD] double word，x86/Intel 风格汇编文本中常表示 32 位内存操作数。
\item[KV] Key/Value，键值状态或键值存储的简写。AI 章节中的 KV cache 另有更具体含义。
\item[CAP] Consistency、Availability、Partition tolerance 的分布式系统权衡框架。
\item[Raft] 一类易理解的共识协议，用 leader、log、term 和 commit index 组织复制日志。
\item[Paxos] 共识协议族，核心是多数派接受和提案编号，常作为理解共识边界的理论入口。
\item[quorum] 法定票数集合，常见实现是多数派，用于避免两个互斥事实同时提交。
\item[leader term] 领导者任期编号，用于区分新旧 leader，防止旧 leader 继续提交。
\item[fencing] 用 term、epoch、lease version 等 token 拒绝旧持有者写外部事实。
\item[protocol version] 协议版本号，用于让新旧节点判断消息格式和语义是否兼容。
\item[magic] 协议帧中的固定识别字节，用来快速发现误读、串流错位或错误文件。
\item[max frame] 协议允许的最大单帧大小，用于限制内存占用和恶意输入风险。
\item[trace 代号] 教学日志或表格里的短代号，例如 \code{T2/A1}，通常表示 task 2 的第 1 次 attempt。
\item[共识] 分布式系统中多个节点在故障下对日志顺序或状态达成一致的协议问题。
\end{description}
