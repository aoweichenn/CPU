\chapter*{全书结构}

本册分为五个部分，围绕同一个大作业展开：在 Linux 本地实现一个 CPU 量化推理引擎。

第一部分是 Linux 上的现代 CPU 性能工程。它先定义贯穿项目和本地 Linux 环境，再解释现代多核心 CPU、缓存、TLB、预取、内存带宽、NUMA、性能模型和可信测量。读者在这里要获得一个判断能力：程序慢，到底更像计算问题、内存问题、同步问题还是测量问题。

第二部分是并发、同步与软件优化。它讲线程、任务切分、锁、原子操作、内存模型、false sharing、SIMD 和缓存友好布局。推理引擎最终一定要处理多线程执行，因此这一部分要先把运行时基础讲清楚，再谈 AI 算子。

第三部分是 AI 基础。它从零解释张量、shape、stride、dtype、参数、激活、模型、推理图、算子、Embedding、MLP、CNN、Transformer 和 Attention。这个部分的目标不是训练模型，而是让没有 AI 背景的读者能看懂推理时到底在算什么、数据怎样流动、算子接口应该怎样设计。

第四部分是算子开发。它把数学公式落到 CPU 内核上：矩阵乘从三重循环到分块和 packing，卷积和归一化从数据流角度分析，Softmax 解释数值稳定性和归约结构。所有算子都服务于贯穿项目，而不是孤立示例。

第五部分是量化推理引擎。它把 Attention、KV Cache、int8 量化、低精度累加、模型文件、推理图执行、benchmark suite 和源码阅读连接起来。最终项目要能在 Linux 本地加载一个教学量化模型，执行推理，输出可验证结果，并报告延迟、吞吐和主要瓶颈。

\begin{keyidea}
全书的结构不是“CPU 知识加 AI 知识”的拼接，而是一条工程链：Linux 机器模型决定运行边界，并发运行时决定多核执行方式，AI 基础决定算子接口，算子内核决定局部性能，量化推理引擎把所有部件接成系统。
\end{keyidea}
