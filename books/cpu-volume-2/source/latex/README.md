# LaTeX 正式书稿工程

这是 `books/cpu-volume-2` 的正式教材工程。第二册主题是计算系统：现代 CPU 硬件原理、多核心并行、高性能计算、多线程、同步、无锁数据结构、并行算法、运行时、异步 I/O 和分布式计算。AI 模型、算子开发和推理引擎放到第三册。

## 构建命令

```bash
make check
make pdf
make epub
make text-count
```

生成文件：

- `main.pdf`
- `main.epub`

## 排版规则

- 版式、字体、代码块、目录层级和 EPUB 约束沿用第一册。
- 目录只保留部和章，章内小节不进入目录。
- 正文使用编号小节串联内容，避免碎片化罗列。
- 代码使用 Maple Mono NL NF CN，保持等宽缩进、语法颜色和行号。
- EPUB 不生成封面页，不包含图片文件或图片标签，不输出 XHTML 表格。

## 写作规则

- 原理先行，再给代码和实验。
- 正文最终目标不少于 50 万单位，统计口径是 `chapters/` 下中文字符加英文单词；代码、命令、反汇编、配置、前言、目录、附录和 LaTeX 标记不计入。50 万只是最低验收线，不是上限；若知识完整性需要，正文可以明显超过 50 万。
- 各章必须按书籍标准扩写，不能用附录、题表、重复段落或模板段落凑数。
- 各章都必须包含高知识密度的机制讲解、工程案例、反例、实验设计、作业和源码阅读入口；不能只给定义和结论。
- 代码示例不能只给一段最终答案。涉及算法或系统结构时，至少要给 baseline、改进版本、关键边界版本，并解释每个版本解决了什么问题、引入了什么代价。
- 贯穿项目是 Compute Systems Engine：从并行归约、有界队列和测量 harness 起步，逐步扩展到线程池、任务图、工作窃取、SPSC/MPMC 队列、背压、分布式 shuffle 和故障注入。每章都要说明本章内容如何落到这个项目。
- 性能结论必须配合硬件拓扑、计算量、内存流量、同步方式和测量边界。
- C++ 示例使用位宽明确的整数类型，例如 `std::int32_t`、`std::int64_t` 和 `std::uint64_t`。
- 不把源码阅读写成函数列表，要围绕硬件机制、数据结构、同步边界、热路径、benchmark 和工程报告展开。
- 项目代码位于 `labs/compute_systems/`，后续扩写要让章节内容能落到这个项目上。
