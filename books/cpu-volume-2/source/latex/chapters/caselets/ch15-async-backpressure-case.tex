\begin{casebox}
\textbf{知识点小案例：异步提交成功不等于业务提交成功。}
特例是把一个 block 写到临时文件。提交异步写请求成功，只说明内核接受了请求；completion 成功，只说明某次写入完成；业务提交还需要 checksum、rename 或 manifest 记录。若把 buffer 在 completion 前归还池子，设备可能还在使用旧内容；若没有高低水位线，in-flight bytes 会把内存打爆。由此推出规律：异步 I/O 必须拆开 submit、completion、validation、commit 和 reclaim。工程上每个请求都要有状态机、owner、offset、remaining、bytes in flight 和取消/关闭规则。
\end{casebox}
