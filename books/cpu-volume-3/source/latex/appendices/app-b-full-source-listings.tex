\section{源码卷怎样对应第三册正文}

\begin{description}
  \item[构建入口与模型资产] 对应本书推理全链路和模型资产章节。先读 CMake、README、tiny model 和 tiny tokenizer，明确哪些目标可构建、哪些 smoke 依赖外部模型。
  \item[公共合同] 对应张量、tokenizer、safetensors、kernel、decoder 和 GPT-2 reference path 的合同。先读头文件，再读实现。
  \item[Tiny MLP 与 int8 kernel] 对应量化、GEMM、packing 和 SIMD 章节。重点看 scalar baseline、packed layout、AVX2 可用性和误差对照。
  \item[Reference decoder 与 GPT-2] 对应 Transformer、KV cache、采样、模型导入和真实 GPT-2 smoke 章节。重点看 trace checkpoint、KV cache 地址、BPE、safetensors dtype 和 HuggingFace 权重名映射。
  \item[工具与测试] 对应 benchmark、7B ledger、serving SLO probe、真实模型 smoke 和工程验收章节。重点看报告字段和 CTest 断言。
\end{description}

如果读者只想复现第三册的代码证据，优先运行：

\begin{lstlisting}[language=bash]
cmake -S books/cpu-volume-3 -B books/cpu-volume-3/build/lcqi-debug -DCMAKE_BUILD_TYPE=Debug
cmake --build books/cpu-volume-3/build/lcqi-debug
ctest --test-dir books/cpu-volume-3/build/lcqi-debug --output-on-failure
\end{lstlisting}

如果读者要审阅完整源码，请从源码卷第一章开始读，不要直接跳到最长的 GPT-2 实现文件。LCQI 的质量取决于构建、资产、合同、实现、工具和测试是否连成一条证据链。
