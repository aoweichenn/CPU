\chapter*{完整源码清单}
\addcontentsline{toc}{chapter}{完整源码清单}

本附录不是节选代码，而是从 \filepath{books/compute-systems-engine-code/labs/reference_pipeline} 直接读入的完整工程源码。读者在 PDF 或 EPUB 中看到的清单，应当和本地可构建工程保持同源；如果源码修改，重建书籍后清单会随之更新。

源码阅读顺序不要从最长的 \filepath{src/reference_pipeline.cpp} 开始。先读公共头文件，确认类型和函数边界；再读实现文件，观察 parser、reference、split 和 merge 如何分层；然后读 CLI，确认命令行没有复制业务逻辑；最后读测试和 benchmark，检查哪些行为已经被固定，哪些只是性能 smoke。
