\chapter*{术语表}

\begin{description}
  \item[Compute Systems Engine] 本卷贯穿项目，用于承接第二册原理卷中的硬件、并发、运行时、I/O 和分布式计算机制。它不是一个单一 demo，而是一组逐步扩展的工程合同、模型、测试和报告。
  \item[reference] 参考实现，用清楚、确定、可复查的方式定义正确结果。reference 可以慢，但不能含糊；优化实现必须先与 reference 对齐，才有资格比较性能。
  \item[oracle] 判定器，用于比较 reference 和待测实现的输出是否满足同一语义。oracle 可以比较精确值，也可以按误差合同比较浮点或统计结果。
  \item[contract] 合同，代码边界的明确约定：输入、输出、错误、前置条件和不承诺行为。合同越早写成类型和测试，后续重构越不容易漂移。
  \item[RecordId] 记录身份，用 input name、generation、byte offset 和 line number 定位一条输入。它解决的是“坏行从哪里来”和“同名输入的哪一代产生了这个结果”。
  \item[InputCase] 一条输入用例，通常包含原始文本、来源和可重放身份。它比裸字符串多了上下文，因此诊断和报告能追溯。
  \item[ReferenceResult] 参考实现产生的结构化输出，用来喂给 oracle 做正确性判定。它包含输入身份、总记录数、accepted、diagnostics、counts 和 checksum。
  \item[ParseDiagnostic] 解析阶段产生的结构化错误事实，包含错误类别、位置和可读说明。它不是普通日志，因为测试会断言它的字段。
  \item[diagnostic] 结构化诊断事实，说明哪条记录为什么被拒绝或哪条边界失败。好的 diagnostic 同时服务机器重放和人类定位。
  \item[checksum] 校验和，把输出内容压成摘要，用于稳定性检查、截断检查和重放对比。checksum 不能替代完整结果，但能快速发现结果漂移。
  \item[manifest] 机器可读清单，记录输入版本、输出位置、校验和、参数和提交事实。manifest 的价值在于发布后仍可复查，而不是只在屏幕上看一次。
  \item[checkpoint] 检查点，保存可恢复状态，让失败后可以从已提交边界继续。checkpoint 要说明覆盖到哪条记录、哪个 epoch、哪些输出已经安全发布。
  \item[attempt] 一次物理尝试。同一个逻辑任务可能因为失败重试而产生多个 attempt；如果报告不区分 attempt，就会把重复执行误当新任务。
  \item[epoch] 版本或轮次编号，用来区分不同时刻的状态和提交边界。内存回收、配置发布和分布式提交都常用 epoch 避免旧状态覆盖新状态。
  \item[watermark] 水位线，表示某个进度之前的数据已经被观察、处理或提交。它常用于流处理、恢复和状态清理，帮助系统判断哪些旧数据可以安全释放。
  \item[quorum] 法定多数或提交多数。分布式系统里，达到 quorum 后才认为某个值有足够副本支持。第 17 章模型用它解释为什么三个副本通常需要两个确认。
  \item[backpressure] 反压。当下游处理速度低于上游生产速度时，系统必须限制提交、阻塞生产者、丢弃低优先级任务或扩容。没有 backpressure，队列会无限增长，最终把内存和延迟一起拖垮。
  \item[idempotent] 幂等，重复执行同一逻辑请求不会产生额外业务结果。重试、manifest、checkpoint 和 RPC 都依赖幂等边界。
  \item[split] 输入切分后的逻辑片段，通常决定调度、重试和 attempt 身份。split 不能破坏记录边界，否则会产生丢行、重行或错行号。
  \item[batch] 热路径里成批处理的一组记录，通常服务缓存局部性、SIMD 或线程分摊。split 偏调度身份，batch 偏执行效率，二者不要混用。
  \item[partial result] 局部结果。每个 split 或 worker 先产出 partial result，再由 merge 合并。partial result 必须保留足够身份和 checksum，才能定位错误来源。
  \item[merge] 合并步骤。它把多个 partial result 变成一个全局结果。正确 merge 不只是相加，还要处理排序、诊断位置、重复 attempt 和 checksum 合同。
  \item[reference pipeline] 本卷第一条代码主线，从 \code{event,value} 输入推到 parser、reference、report、manifest、split 和 merge。
  \item[chapter model] 每章机制的小模型。它把原理卷里的概念变成输入字段、输出字段和测试断言，例如 TLB miss、lane utilization、quorum size 或 replay range。
  \item[probe] 探针程序。它运行一个模型或机制并打印稳定输出，方便在书、测试和命令行之间对照。
  \item[PMU: Performance Monitoring Unit] 性能监控单元。它提供硬件事件计数，是判断 SIMD、cache、branch 和 pipeline 行为的重要证据来源。
  \item[SIMD: Single Instruction, Multiple Data] 单指令多数据。一条指令同时处理多个元素。实践代码会同时记录 vector iterations 和 scalar tail，避免只看是否“用了向量指令”。
  \item[ILP: Instruction-Level Parallelism] 指令级并行。独立指令越多，CPU 越可能重叠执行；依赖链、分支和内存等待会降低 ILP。
  \item[TLB: Translation Lookaside Buffer] 地址转换后备缓冲。它缓存虚拟页到物理页的映射；TLB miss 和 cache miss 位置不同、优化手段也不同。
  \item[NUMA: Non-Uniform Memory Access] 非一致内存访问。本地内存和远端内存成本不同，线程绑核、内存分配和共享数据布局都会影响 NUMA 行为。
  \item[roofline] 性能上限模型，用 FLOPs、字节量、内存带宽和峰值计算能力估计程序受计算限制还是受内存限制。它不是最终性能报告，只是定位瓶颈的框架。
  \item[MPMC: Multiple Producer, Multiple Consumer] 多生产者多消费者队列。它必须定义容量、关闭、等待、唤醒和重复消费边界。
  \item[SPSC: Single Producer, Single Consumer] 单生产者单消费者队列。常见于 ring buffer，模型重点是 full、empty、push 和 pop 的边界条件。
  \item[CAS: Compare-And-Swap] 比较并交换。无锁结构常用它尝试发布新状态；CAS 失败表示状态已被别人修改，需要重新读取和决策。
  \item[ABA Problem] ABA 问题。共享值从 A 变 B 又变回 A，观察者只看当前值会误判没有变化。解决方式包括版本号、hazard pointer、epoch 或禁止过早复用节点。
  \item[RCU: Read-Copy-Update] 读-复制-更新。读路径尽量轻，写路径发布新版本并延迟回收旧版本。它帮助解释为什么 epoch reclamation 要知道哪些读者还在旧视图里。
  \item[futex: Fast Userspace Mutex] Linux 快速用户态互斥机制。无竞争时在用户态完成，有竞争时通过内核 wait/wake 阻塞和唤醒。
  \item[eventfd] Linux 事件计数文件描述符。它可被 \code{epoll} 监听，适合把跨线程完成事件接入统一事件循环。
  \item[epoll] Linux I/O 多路复用接口。它报告文件描述符 ready，但 ready 不等于业务数据已经完全处理；代码仍要 drain、解析和提交。
  \item[future/promise] C++ 异步结果通道。promise 负责设置结果，future 负责等待和读取结果。本卷把它用于解释任务运行时里的等待边界和同池等待风险。
  \item[atomic wait] C++20 原子等待机制。线程可以等待原子值变化，而不是一直自旋。实践中要配合正确的 memory order 和 notify。
  \item[work stealing] 工作窃取。空闲 worker 从其他 worker 的队列偷任务，改善负载不均。模型会记录 stolen tasks、completed tasks 和队列深度。
  \item[CPU: Central Processing Unit] 中央处理器。本卷先把实现、测试和 benchmark 落在本地 CPU 环境。
  \item[C++20] C++ 标准版本名。本卷默认使用的语言和标准库能力边界。
  \item[CMake] C++ 工程构建系统，用于组织 target、测试、benchmark 和工具程序。
  \item[GTest: GoogleTest] C++ 单元测试框架。它固定行为边界，让正文结论能被代码验证。
  \item[GMock: GoogleMock] GoogleTest 的 mock 框架，用于控制依赖和断言交互。
  \item[Google Benchmark] C++ benchmark 库。本卷用它做性能入口，但 benchmark 输出必须配合输入规模、构建类型和正确性校验解释。
  \item[CLI: Command-Line Interface] 命令行接口，用参数、标准输入输出和退出码表达工具行为。
  \item[CI: Continuous Integration] 持续集成，用自动构建、测试和检查保护每次提交。
  \item[JSON: JavaScript Object Notation] 结构化文本格式，适合保存配置、manifest 和报告。
  \item[CSV: Comma-Separated Values] 逗号分隔文本表格，适合保存实验结果和统计数据。
  \item[I/O: Input/Output] 输入输出。异步 I/O 中 submit、completion 和 commit 不是同一件事。
  \item[EOF: End Of File] 输入结束。读取代码要区分 EOF、短读和错误。
  \item[offset] 输入中的字节偏移，用于机器重放和精确定位。
  \item[line number] 输入文本中的行号，用于人阅读诊断和快速定位错误。
\end{description}
