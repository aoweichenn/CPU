\chapter*{术语表}

\begin{description}
  \item[Compute Systems Engine] 本卷贯穿项目，用于承接第二册原理卷中的硬件、并发、运行时、I/O 和分布式计算机制。
  \item[reference] 参考实现，用清楚、确定、可复查的方式定义正确结果。
  \item[oracle] 判定器，用于比较 reference 和待测实现的输出是否满足同一语义。
  \item[contract] 合同，代码边界的明确约定：输入、输出、错误、前置条件和不承诺行为。
  \item[RecordId] 记录身份，用 input name、generation、byte offset 和 line number 定位一条输入。
  \item[diagnostic] 结构化诊断事实，说明哪条记录为什么被拒绝或哪条边界失败。
  \item[checksum] 校验和，把输出内容压成摘要，用于稳定性检查、截断检查和重放对比。
  \item[CPU] Central Processing Unit，中央处理器。本卷先把实现、测试和 benchmark 落在本地 CPU 环境。
  \item[UI] User Interface，用户界面。实践卷里主要指用户看到的诊断、报告和工具输出。
  \item[C++20] C++ 标准版本名。本卷默认使用的语言和标准库能力边界。
  \item[CMake] C++ 工程常用构建系统，用于组织 target、测试、benchmark 和工具程序。
  \item[\code{std::}] C++ 标准库命名空间前缀。\code{std::vector} 表示拥有元素的动态数组，\code{std::string} 表示字节字符串，\code{std::map} 表示按键排序的关联容器，\code{std::uint64_t} 表示固定 64 位无符号整数。
  \item[InputCase] 第一章示例中的一条输入用例，通常包含原始文本、来源和可重放身份。
  \item[ReferenceResult] 参考实现产生的结构化输出，用来喂给 oracle 做正确性判定。
  \item[ParseDiagnostic] 解析阶段产生的结构化错误事实，包含错误类别、位置和可读说明。
  \item[SIMD] Single Instruction, Multiple Data，一条指令处理多个数据。
  \item[I/O] Input/Output，输入输出。异步 I/O 中 submit、completion 和 commit 不是同一件事。
  \item[EOF] End Of File，输入结束。读取代码要区分 EOF、短读和错误。
  \item[CLI] Command-Line Interface，命令行接口，用参数、标准输入输出和退出码表达工具行为。
  \item[CI] Continuous Integration，持续集成，用自动构建、测试和检查保护每次提交。
  \item[JSON] JavaScript Object Notation，结构化文本格式，适合保存配置、manifest 和报告。
  \item[CSV] Comma-Separated Values，逗号分隔文本表格，适合保存实验结果和统计数据。
  \item[manifest] 机器可读清单，记录输入版本、输出位置、校验和、参数和提交事实。
  \item[checkpoint] 检查点，保存可恢复状态，让失败后可以从已提交边界继续。
  \item[attempt] 一次物理尝试。同一个逻辑任务可能因为失败重试而产生多个 attempt。
  \item[epoch] 版本或轮次编号，用来区分不同时刻的状态和提交边界。
  \item[MessageBus] 消息总线，用于组件之间传递任务、完成事件和控制消息。
  \item[split] 输入切分后的逻辑片段，通常决定调度、重试和 attempt 身份。
  \item[batch] 热路径里成批处理的一组记录，通常服务缓存局部性、SIMD 或线程分摊。
  \item[idempotent] 幂等，重复执行同一逻辑请求不会产生额外业务结果。
  \item[offset] 输入中的字节偏移，用于机器重放和精确定位。
  \item[line number] 输入文本中的行号，用于人阅读诊断和快速定位错误。
\end{description}
